JavaScript del Lado del Cliente y del Lado del Servidor

¿Qué es JavaScript?

JavaScript es un lenguaje de programación versátil que se puede ejecutar tanto en el lado del cliente (cliente-side) como en el lado del servidor (server-side). A continuación, exploraremos las diferencias y los usos de cada uno.

JavaScript del Lado del Cliente

El JavaScript del lado del cliente se ejecuta en el navegador del usuario. Es principalmente utilizado para crear interfaces de usuario interactivas y dinámicas. Algunas de sus características y usos más comunes incluyen:

  • Manipulación del DOM
  • Eventos de Usuario
  • Validación de Formularios
  • AJAX
  • Animaciones
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>JavaScript del Lado del Cliente</title>
</head>
<body>
    <h1 id="titulo">Hola, Mundo!</h1>
    <button onclick="cambiarTexto()">Haz clic aquí</button>

    <script>
        function cambiarTexto() {
            document.getElementById("titulo").innerText = "Has hecho clic en el botón!";
        }
    </script>
</body>
</html>

JavaScript del Lado del Servidor

El JavaScript del lado del servidor se ejecuta en el servidor, fuera del entorno del navegador. Node.js es el entorno de ejecución más popular para JavaScript del lado del servidor. Este tipo de JavaScript se utiliza para crear aplicaciones web, manejar bases de datos, gestionar la lógica del servidor y más.

  • Gestión de Bases de Datos
  • Autenticación y Autorización
  • Lógica del Negocio
  • API REST
  • Procesamiento en Tiempo Real
// app.js
const http = require('http');

const server = http.createServer((req, res) => {
    res.statusCode = 200;
    res.setHeader('Content-Type', 'text/plain');
    res.end('Hola, Mundo!\n');
});

const PORT = 3000;
server.listen(PORT, () => {
    console.log(`Servidor corriendo en http://localhost:${PORT}/`);
});

Resumen de las Diferencias

Característica Lado del Cliente Lado del Servidor
Ejecutado en Navegador del usuario Servidor (ej. Node.js)
Propósito principal Interactividad y manipulación de la UI Lógica de negocio, gestión de bases de datos, APIs
Acceso al DOM Sí No
Ejemplos comunes Validación de formularios, animaciones, AJAX Autenticación, autorización, interacción con bases de datos
Entornos comunes Todos los navegadores modernos Node.js, Deno

Control de Versiones con Git

Introducción a Git

Git es un sistema de control de versiones distribuido que permite a múltiples desarrolladores trabajar en un proyecto de manera simultánea y eficiente. Proporciona herramientas para rastrear cambios, colaborar en el código y mantener un historial completo de todas las modificaciones realizadas en el proyecto.

Conceptos Básicos de Git

  • Repositorio (Repository): Un directorio que contiene el historial completo de los archivos de un proyecto, incluyendo todas sus versiones y cambios.
  • Commit: Un registro de cambios que se ha realizado en el repositorio. Cada commit tiene un identificador único y puede incluir un mensaje descriptivo.
  • Branch (Rama): Una línea de desarrollo independiente. Permite trabajar en diferentes características o correcciones de errores sin afectar la rama principal.
  • Merge: La acción de combinar cambios de diferentes ramas en una sola rama.
  • Clone: Una copia de un repositorio que se descarga desde un servidor remoto a una máquina local.
  • Push: La acción de enviar commits desde el repositorio local a un repositorio remoto.
  • Pull: La acción de actualizar el repositorio local con los cambios realizados en el repositorio remoto.
  • Staging Area: Una área temporal donde se colocan los cambios que se desean incluir en el próximo commit.

Comandos Básicos de Git

Inicializar un Repositorio

Para crear un nuevo repositorio, usa el comando git init en el directorio del proyecto.

git init

Clonar un Repositorio

Para clonar un repositorio existente desde un servidor remoto, usa el comando git clone seguido de la URL del repositorio.

git clone https://github.com/usuario/repositorio.git

Agregar Cambios

Para agregar archivos al staging area, usa el comando git add.

git add nombre_del_archivo
# Para agregar todos los archivos cambiados
git add .

Crear un Commit

Para crear un commit con los cambios agregados al staging area, usa el comando git commit. 

git commit -m "Descripción de los cambios"

Ver el Historial de Commits

Para ver el historial de commits, usa el comando git log.

git log

Crear y Cambiar Ramas

Para crear una nueva rama, usa el comando git branch.

git branch nombre_de_la_rama

Para cambiar a una rama existente, usa el comando git checkout.

git checkout nombre_de_la_rama

O para crear y cambiar a una nueva rama al mismo tiempo:

git checkout -b nombre_de_la_rama

Fusionar Ramas

Para fusionar una rama en la rama actual, usa el comando git merge.

git merge nombre_de_la_rama

Enviar Cambios a un Repositorio Remoto

Para enviar los commits locales a un repositorio remoto, usa el comando git push.

git push origen nombre_de_la_rama

Actualizar el Repositorio Local

Para actualizar el repositorio local con los cambios del repositorio remoto, usa el comando git pull.

git pull origen nombre_de

Semántica en HTML

¿Qué es la semántica en HTML?

La semántica son etiquetas agregadas en la evolución de HTML4 a HTML5. Estas etiquetas cumplen funciones que ya eran cumplidas en HTML4 pero con el plus de que ahora en HTML5 permiten tener un código más legible con un estándar de nombres como <header>, mejorando la optimización del código y su estructura. Además, estas etiquetas facilitan al SEO (Searching engine system / sistema de búsqueda) ser más eficiente.

Las etiquetas semánticas también están preparadas para adaptarse a dispositivos móviles, haciendo el contenido más accesible y fomentando mejores prácticas de desarrollo. Esto lleva a un código más limpio y comprensible.

Ventajas de usar etiquetas semánticas en HTML5

  • Mejora la comprensión de los usuarios sobre el código.
  • Claridad del código para tener una mejor estructura para los desarrolladores.
  • Facilita al SEO (motor de búsqueda) comprender la estructura del código y mejorar el posicionamiento de los resultados.
  • Mantenimiento del código y actualización más fácil.
  • Compatibilidad con dispositivos futuros como dispositivos móviles.

Desventajas de las etiquetas semánticas

  • Puede ser difícil para los programadores veteranos adaptarse a estos cambios, teniendo una curva de aprendizaje inicial.
  • Problemas de compatibilidad con navegadores antiguos.
  • Uso incorrecto de las etiquetas puede resultar en una estructura rara o difícil de comprender.

Principales etiquetas semánticas en HTML5

<header>

Define la cabecera de un documento o una sección de cabecera.

<nav>

Esta etiqueta crea una sección de barra de navegación donde puedes colocar links.

<main>

Define el contenido principal del documento.

<section>

Hace una separación de sección del documento.

<article>

Define un artículo independiente y autocontenido.

<aside>

Define contenido relacionado pero independiente del contenido principal, como barras laterales.

<footer>

Define el pie de página de un documento o una sección.

<address>

Define información de contacto del autor o propietario de un documento.

<h1> - <h6>

Ayudan a crear los títulos para la página. Ejemplo: <h1>Soy un título</h1>

<time>

Define fechas o períodos de tiempo.

<figure>

Define contenido independiente, como imágenes, diagramas o tablas.

<details>

Define los detalles adicionales que el usuario puede mostrar u ocultar.

<summary>

Hace el encabezado visible para el contenido oculto dentro de un elemento <details>.

<figcaption>

Define una leyenda para un elemento <figure>.

<mark>

Hace texto destacado o resaltado.

<progress>

Define una barra de progreso.

<meter>

Define una medida dentro de un rango conocido.

<wbr>

Define un posible punto de quiebre en una palabra.

<kbd>

Define entrada de teclado.

<code>

Define una pieza de código.

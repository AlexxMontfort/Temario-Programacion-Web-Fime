Terminología de Git

Repositorio (Repository)

Un repositorio es una estructura de datos que almacena el historial completo de los cambios realizados en un proyecto, incluyendo todos los archivos y sus versiones. Puede ser local (en tu máquina) o remoto (en un servidor).

Commit

Un commit es una instantánea del estado actual del proyecto. Representa un conjunto de cambios que se han registrado en el repositorio. Cada commit tiene un identificador único (hash) y puede incluir un mensaje descriptivo.

Rama (Branch)

Una rama es una línea de desarrollo independiente. Permite trabajar en diferentes funcionalidades o correcciones de errores sin afectar la rama principal (usualmente llamada main o master).

Merge

El merge es el proceso de combinar cambios de diferentes ramas en una sola rama. Permite integrar el trabajo de múltiples desarrolladores.

Clone

Clonar un repositorio significa crear una copia exacta de un repositorio remoto en tu máquina local. Incluye todo el historial del proyecto.

Push

El push es el proceso de enviar commits desde el repositorio local a un repositorio remoto. Es como publicar tus cambios para que otros desarrolladores puedan verlos.

Pull

El pull es el proceso de actualizar tu repositorio local con los cambios realizados en el repositorio remoto. Incluye dos pasos: fetch (descargar los cambios) y merge (combinar los cambios).

Fork

Un fork es una copia de un repositorio en tu propia cuenta de GitHub (o cualquier otra plataforma de alojamiento de Git). Permite trabajar en un proyecto sin afectar el repositorio original.

Staging Area

La staging area es una área temporal donde se colocan los cambios que se desean incluir en el próximo commit. Permite revisar y agrupar cambios antes de confirmarlos.

Index

El índice es otra forma de referirse a la staging area. Es donde se preparan los cambios para ser committeados.

HEAD

HEAD es un puntero que apunta al commit actual en el que estás trabajando. Usualmente apunta a la rama activa.

Remote

Un remote es una versión del repositorio que está alojada en un servidor. Los remotes permiten colaborar con otros desarrolladores.

Origin

Origin es el nombre por defecto que Git da al repositorio remoto cuando clonas un repositorio por primera vez. Es el punto de referencia para sincronizar cambios.

Tag

Un tag es una referencia a un punto específico en la historia del repositorio. Se usa comúnmente para marcar versiones de software (por ejemplo, v1.0).

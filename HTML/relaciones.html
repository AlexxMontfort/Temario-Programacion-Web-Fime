Relaciones en Bases de Datos

Relación Uno a Uno (1-1)

Una relación uno a uno (1-1) significa que a cada fila de la tabla A le corresponde exactamente una fila en la tabla B, y viceversa.

Ejemplo:

Personas y direcciones, donde cada persona tiene una única dirección.

CREATE TABLE Personas (
    id_persona INT PRIMARY KEY,
    nombre VARCHAR(100),
    edad INT
);

CREATE TABLE Direcciones (
    id_direccion INT PRIMARY KEY,
    id_persona INT UNIQUE,
    calle VARCHAR(100),
    ciudad VARCHAR(100),
    país VARCHAR(100),
    FOREIGN KEY (id_persona) REFERENCES Personas(id_persona)
);

Relación Uno a Muchos (1-N)

Una relación uno a muchos (1-N) significa que a cada fila de la tabla A le pueden corresponder muchas filas en la tabla B, pero cada fila de la tabla B solo puede corresponder a una fila en la tabla A.

Ejemplo:

Autores y libros, donde un autor puede escribir muchos libros, pero cada libro tiene un solo autor.

CREATE TABLE Autores (
    id_autor INT PRIMARY KEY,
    nombre VARCHAR(100),
    nacionalidad VARCHAR(50)
);

CREATE TABLE Libros (
    id_libro INT PRIMARY KEY,
    titulo VARCHAR(100),
    id_autor INT,
    FOREIGN KEY (id_autor) REFERENCES Autores(id_autor)
);

Relación Muchos a Muchos (N-N)

Una relación muchos a muchos (N-N) significa que a cada fila de la tabla A le pueden corresponder muchas filas en la tabla B, y viceversa.

Ejemplo:

Estudiantes y cursos, donde un estudiante puede inscribirse en muchos cursos, y cada curso puede tener muchos estudiantes.

CREATE TABLE Estudiantes (
    id_estudiante INT PRIMARY KEY,
    nombre VARCHAR(100),
    edad INT
);

CREATE TABLE Cursos (
    id_curso INT PRIMARY KEY,
    titulo VARCHAR(100),
    descripcion TEXT
);

CREATE TABLE Estudiantes_Cursos (
    id_estudiante INT,
    id_curso INT,
    PRIMARY KEY (id_estudiante, id_curso),
    FOREIGN KEY (id_estudiante) REFERENCES Estudiantes(id_estudiante),
    FOREIGN KEY (id_curso) REFERENCES Cursos(id_curso)
);
